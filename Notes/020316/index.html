Notes for Wednesday, Feb 3. Scalable Vector Graphics

SVG is an XML specification of vector graphics. Visual elements are represented as tags, which have styles and attributes. Here's a reference on SVG that includes the basic shape elements along with filters (blur! shadows!), animations, and transformations.
1. Polylines. SVG interprets the points attribute as a string containing at least one pair of x,y coordinates. Coordinates can be integers or floating point numbers. Commas between coordinates are optional, but helpful for (human) readers. The order of points determines where the line goes and what the shape of the fill looks like. Try reversing the middle two coordinates.
2. Paths. This is the most complicated SVG element, but also one of the most useful. It contains a tiny programming language. Uppercase commands specify absolute coordinates, lowercase commands specify offsets from the current position, wherever that is. This is not unlike the LOGO programming language.
3. Transformations are another useful (but a bit tricky) tool. Here's a reference. Any SVG element can have a transform attribute. To apply the same transform on multiple tags nest them inside a <g> tag. Use scale(factor) to stretch a shape. Use translate(x,y) to move a shape. Use rotate(degrees) to rotate a shape. Your transform attribute can contain more than one of these. The order of operations matters. Guessing what will happen can be difficult. Try several combinations and see what you get.
